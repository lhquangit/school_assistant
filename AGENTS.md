# Project Agents

This repository keeps project-local Codex skills in `.codex/skills/`.

Before starting substantial work, read the relevant skill file and follow it:

- Feature planning or unclear requirements:
  `.codex/skills/grill-with-docs/SKILL.md`
- Test-first implementation:
  `.codex/skills/tdd/SKILL.md`
- Debugging bugs or regressions:
  `.codex/skills/diagnose/SKILL.md`
- Turning discussion into a PRD:
  `.codex/skills/to-prd/SKILL.md`
- Breaking a PRD or plan into vertical-slice issues:
  `.codex/skills/to-issues/SKILL.md`
- Triage workflow for incoming issues:
  `.codex/skills/triage/SKILL.md`
- Architecture review and deepening opportunities:
  `.codex/skills/improve-codebase-architecture/SKILL.md`
- High-level codebase mapping for unfamiliar areas:
  `.codex/skills/zoom-out/SKILL.md`

Use the project's domain language consistently. When domain terms or architectural decisions are clarified, capture them in `CONTEXT.md` and `docs/adr/`.
